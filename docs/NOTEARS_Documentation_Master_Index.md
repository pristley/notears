## 📖 Glossary of Key Terms

**Acyclicity Constraint** `h(W)`
- Mathematical function h(W) = tr(exp(W ⊙ W)) - d
- h(W) = 0 if and only if graph is acyclic
- Core innovation of NOTEARS
- See: Implementation_Guide.md, Section 1.2

**DAG (Directed Acyclic Graph)**
- Graph where edges flow in one direction (no cycles)
- Represents causal relationships
- Central object of study in causal discovery

**Matrix Exponential** `exp(W)`
- Mathematical function applied to matrices
- Computed via scaling-and-squaring for numerical stability
- O(d³) complexity; computational bottleneck
- See: Implementation_Guide.md, Prompt 3

**Hadamard Product** `⊙`
- Element-wise multiplication of matrices
- Used in h(W) = tr(exp(W ⊙ W)) - d
- See: Implementation_Guide.md, Prompt 4

**L-BFGS (Limited-memory Broyden–Fletcher–Goldfarb–Shanno)**
- Quasi-Newton optimization algorithm
- Approximates Hessian with limited memory
- Used to solve primal subproblems
- See: Implementation_Guide.md, Prompt 10

**Augmented Lagrangian**
- Optimization technique for constrained problems
- Converts constrained problem to sequence of unconstrained subproblems
- Enables use of black-box solvers (L-BFGS)
- See: Implementation_Guide.md, Section 2

**Structural Hamming Distance (SHD)**
- Metric: number of operations (add/delete/reverse edge) to convert est. to true DAG
- Common metric for evaluating DAG learning
- Lower is better
- See: Quick_Reference.md, Section 8

**Faithfulness Assumption**
- Assumes conditional independencies match graph structure exactly
- PC algorithm requires this; NOTEARS does not
- Often violated in practice
- See: Analysis_and_Comparison.md, Dimension 6

**Identifiability / Equivalence Class**
- Multiple DAGs may encode same conditional independence distribution
- NOTEARS recovers DAG up to equivalence class (like GES)
- LiNGAM may identify unique direction
- See: Analysis_and_Comparison.md, Section II.4

**SEM (Structural Equation Model)**
- Statistical model for causal relationships
- NOTEARS assumes linear SEM: X_j = w_j^T X + ε_j
- See: Implementation_Guide.md, Section 1.1

## ❓ FAQ (Frequently Asked Questions)

**Q: Should I use NOTEARS for my problem?**
A: Check decision tree in Quick_Reference.md Section 2, or Decision Matrix in Analysis_and_Comparison.md Section V.

**Q: How do I choose λ (L₁ regularization)?**
A: See Quick_Reference.md Section 4 "Lambda Selection Strategy". Short answer: cross-validation or BIC-style scaling based on n and d.

**Q: My optimization didn't converge. What do I do?**
A: See Quick_Reference.md Section 6, Issue 1. Common causes: λ too small, poor initialization, data quality issues.

**Q: How many edges should I expect?**
A: Depends on domain. Biological networks: ~1 edge per node. See Analysis_and_Comparison.md Case Studies for domain-specific guidance.

**Q: Can NOTEARS handle nonlinear relationships?**
A: No. It assumes linear SEM. For nonlinear data, consider NOTEARS-G, GOLEM, or other methods. See Analysis_and_Comparison.md Limitations.

**Q: Is NOTEARS faster than GES?**
A: Comparable per iteration. NOTEARS may get better solutions faster due to global updates. See benchmarks in Quick_Reference.md Section 5.

**Q: What if I have categorical variables?**
A: NOTEARS assumes continuous. Use NOTEARS-G (2021 extension) for mixed data. See Analysis_and_Comparison.md for alternatives.

**Q: How do I validate results if I don't have ground truth?**
A: See Quick_Reference.md Section 9 "If Ground Truth NOT Available". Options: stability assessment, domain expert validation, comparison to baselines.

**Q: Can NOTEARS handle confounding?**
A: No. Assumes no unmeasured confounders. This is shared by all DAG learning methods. See Analysis_and_Comparison.md Case Study 5.

**Q: Should I run NOTEARS multiple times?**
A: Yes, recommended. Use multiple random seeds (or deterministic initialization) and report stable edges. See Quick_Reference.md Pitfall 10.

**Q: How do I compare NOTEARS to other methods fairly?**
A: Use same hyperparameters, same data preprocessing, same metrics. See Analysis_and_Comparison.md Section IV for systematic comparisons.
